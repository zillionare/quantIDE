# 发现与研究

## 规范要求 (.dev/specs/02-layout-nav-style.md)
1. **总体布局**: Header + Sidebar + Main 三栏布局
2. **样式与主题色**: 
   - 主题色: `#e41815`，作为背景时字体用白色，该颜色不再作为 Header 等大面积区域的背景使用（可使用白色等）。
   - 按钮: 背景 `#fcfcfc`，前景 `#e41815` 或 `#2c3030`。
   - 图标: 不使用 emoji 和 svg，使用统一的 icon 集。
3. **Header 设计**:
   - 包含的元素及顺序：Brand("匡醍量化") -> 空白 -> 一级导航 -> alarm（显示为 alarm (bell) 图标，有未读消息提示覆盖层） -> 用户头像
   - 一级菜单：策略(`/strategy`), 系统维护(`/system`), 实盘(`/live`), 仿真(`/papertrade`)
4. **头像菜单**:
   - 仅显示头像本身，点击下拉菜单图标时，展现的下拉菜单居中对齐展开
   - 菜单项：(icon) 重设密码, (icon) 退出登录
5. **Sidebar设计**:
   - accordion 风格的多级菜单
   - 第一级如果无关联路由，点击时展开/隐藏子菜单
   - Sidebar 变化时默认显示第一个菜单项的内容
