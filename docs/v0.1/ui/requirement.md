# UI 需求文档

## 概述

本文档记录了 PyQMT 交易系统的 UI 需求，包括首页（home.html）和实盘交易页面（livetrade.html）的完整功能需求和设计规范。

---

## 一、首页（home.html）

### 1.1 页面布局

- **整体布局**：Header + Sidebar + Main Content
- **最大宽度**：1400px，居中显示
- **主题**：支持亮色/暗色模式

### 1.2 顶部导航栏（Header）

- Logo + 品牌名称（匡醍）
- 一级导航：首页、交易、行情、资讯、风控
- 右侧：通知图标、用户头像

### 1.3 左侧菜单栏（Sidebar）

- 宽度：64px（收起状态）
- 菜单项：首页、实盘交易、模拟交易、策略交易、行情、资讯、风控

### 1.4 资产信息条

- 展示形式：水平信息条（非卡片式）
- 显示内容：
  - 总资产（单位：万，保留2位小数）
  - 现金
  - 现金比
  - 市值
  - 仓位

### 1.5 持仓明细表

- 列：股票代码、股票名称、持仓数量、可卖数量、成本价、現价、浮动盈亏、盈亏比、AI预警
- 功能：支持排序、筛选

### 1.6 当日委托表

- 列：委托时间、股票代码、股票名称、方向（买入/卖出）、价格、数量、状态

## 二、实盘交易页面（livetrade.html）

### 2.1 页面布局

- **整体布局**：Header + Sidebar + Main Content
- **最大宽度**：1400px，居中显示
- **主题**：支持亮色/暗色模式
- **面包屑**：已移除

### 2.2 账户连接状态

- 显示 QMT 实盘连接状态
- 支持"已连接"和"已断开"两种状态
- 断开时显示重连按钮

### 2.3 资产信息条

- 同首页设计，水平信息条展示

### 2.4 下单面板（三列布局）

```
┌────────────────┬────────┬────────────────┐
│   下单输入区   │ 价格   │   候选股票池   │
│    (2/5)      │ 快捷   │    (2/5)       │
│               │ (1/5)  │                │
└────────────────┴────────┴────────────────┘
```

#### 2.4.1 左列：下单输入区（2/5）

| 序号 | 功能 | 说明 |
|------|------|------|
| a | 股票代码输入框 | placeholder: "输入股票代码、名称或拼音" |
| b | 股票名称显示 | 输入/选择后显示股票名称 |
| c | 价格输入区 | 显示实时价格，可手动输入 |
| d | 委买金额输入框 | 单位：万元 |
| e | 仓位选择 | 全仓、1/2、1/3、1/4 |
| f | 买入/卖出按钮 | 红色买入、绿色卖出 |

#### 2.4.2 中间列：价格快捷输入区（1/5）

三列布局：

| 列 | 内容 | 按钮样式 |
|----|------|----------|
| 第一列 | +1% 到 +10% | 灰色按钮 |
| 第二列 | -10% 到 -1% | 灰色按钮 |
| 第三列 | MA5, MA10, MA20, MA30, MA60, MA120 | 蓝色按钮 |

#### 2.4.3 右列：候选股票池（2/5）

- 搜索框
- 股票列表：显示股票代码、名称、实时价格、涨跌幅
- 点击可选中股票

### 2.5 功能需求

#### 2.5.1 股票搜索

支持以下方式搜索股票：
- 股票代码（如：000001、600000）
- 股票名称（如：平安银行、贵州茅台）
- 拼音简称（如：pingan、maotai）

#### 2.5.2 价格调整

- **手动输入**：直接输入价格
- **百分比调整**：+1% ~ +10%、-1% ~ -10%
- **MA均线调整**：MA5、MA10、MA20、MA30、MA60、MA120

#### 2.5.3 仓位选择

| 按钮 | 含义 | 计算方式 |
|------|------|----------|
| 全仓 | 全部资金 | 可用资金 × 100% |
| 1/2 | 半仓 | 可用资金 × 50% |
| 1/3 | 三分之一仓 | 可用资金 × 33.3% |
| 1/4 | 四分之一仓 | 可用资金 × 25% |

#### 2.5.4 订单提交

- 买入/卖出按钮
- 提交前确认对话框
- 显示：股票名称、代码、价格、金额、数量

## 三、技术规范

### 3.1 技术栈

- **UI 框架**：Tailwind CSS 3.4.17
- **UI 组件库**：Franken UI 2.0.0
- **图标**：Heroicons / SVG
- **JavaScript**：原生 JS（ES6+）

### 3.2 响应式设计

- 桌面端优先设计
- 最大宽度：1400px
- 暗色模式支持

### 3.3 颜色规范

| 用途 | 颜色 |
|------|------|
| 买入按钮 | bg-red-600 |
| 卖出按钮 | bg-green-600 |
| 涨停/高价 | text-red-600 |
| 跌停/低价 | text-green-600 |
| MA均线按钮 | bg-blue-100 |

### 3.4 数据展示规范

- 金额单位：万元（保留2位小数）
- 价格单位：元（保留2位小数）
- 数量单位：股（整手交易，100股=1手）

---

## 四、交互流程

### 4.1 买入流程

1. 输入/选择股票代码 → 自动显示股票名称和实时价格
2. 调整价格 → 手动输入或点击快捷按钮
3. 输入金额 → 输入万元或点击仓位按钮自动计算
4. 点击"买入"按钮
5. 确认对话框显示订单详情
6. 确认提交

### 4.2 卖出流程

同买入流程，点击"卖出"按钮即可。
