# UI 设计文档

## 概述

本文档记录了 PyQMT 交易系统的 UI 设计与思考，重点阐述设计理念、逻辑和考量。页面原型可直接参考 `home.html` 和 `livetrade-default.html`。

---

## 一、设计理念

### 1.1 核心原则

- **效率优先**：交易操作追求快速、精准，减少不必要的点击和思考
- **信息直观**：关键数据一目了然，无需复杂理解
- **操作流畅**：模拟专业交易终端的操作习惯，降低学习成本

### 1.2 用户场景

本系统面向**日内交易者**和**短线交易者**，他们需要：
- 快速下单（选股 → 价格 → 数量 → 确认）
- 实时行情（价格变动直接影响决策）
- 便捷的价格调整工具（无需心算百分比）

## 二、设计考量

### 2.1 为什么选择"现价"作为快捷价格的基准？

**选项对比**：

| 基准 | 优点 | 缺点 |
|------|------|------|
| 现价 | 所见即所得，用户直观 | 盘中波动大时参考价值降低 |
| 昨收价 | 稳定，不受盘中波动影响 | 与当前价格有gap，用户需要心算 |

**最终选择**：**现价**

**理由**：
1. **所见即所得**：用户看到的价格就是计算的基准，符合直觉
2. **适合短线操作**：日内交易者关注的是"如果我现在买，还能以什么价格买入"
3. **可手动干预**：用户可以先修改价格输入框，再按百分比调整，灵活性高

**边界处理**：
- 未开盘时：现价 = 昨收价（系统自动处理，用户无感知）

### 2.2 为什么快捷价格按钮是这些值？

**按钮设计**：

```
┌─────────────┬─────────────┬─────────────┐
│ 1%   2%    │ 跌停  -5%   │ MA5   MA10  │  ← 常用
│ 3%   4%    │ -4%  -3%   │ MA20  MA30  │
│ 5%   涨停   │ -2%  -1%   │ MA60  MA120 │
├─────────────┴─────────────┴─────────────┤
│ 快捷价格                    实时: 13.80 │
└─────────────────────────────────────────┘
```

**设计逻辑**：
- **1%-5%**：最常用的涨幅档位，覆盖大多数情况
- **涨停/+10%**：特殊场景，一键直达
- **-1%~-5%**：常见跌幅档位
- **跌停/-10%**：特殊场景，一键直达
- **MA 均线**：技术分析派常用，一键计算均线附近价格
- **实时价格**：放在标题行右侧，查看当前股价无需额外操作

### 2.3 为什么是三列布局？

```
┌────────────────┬────────┬────────────────┐
│   下单输入区   │ 价格   │   候选股票池   │
│    (2/5)      │ 快捷   │    (2/5)       │
│               │ (1/5)  │                │
└────────────────┴────────┴────────────────┘
```

**设计理由**：
- **左侧（2/5）**：下单是核心功能，需要最大空间
- **中间（1/5）**：价格调整是高频操作，但占用空间小
- **右侧（2/5）**：候选股票池提供快速选股，重要性仅次于下单

### 2.4 为什么输入框采用"标签 + 输入框"纵向对齐？

```
代码      [输入框................................]
股票名称  [显示区................................]
价格      [输入框....................] (实时: 0.00)
金额      [输入框............................] 万元
```

**理由**：
- 标签与输入框一一对应，视觉上更整齐
- 快捷价格区的实时价格已移到按钮下方，价格行的"(实时: xxx)"可移除，简化布局

## 三、账号体系设计

### 3.1 设计理念

- **统一入口**：用户通过一个界面管理所有账号（实盘 + 仿真）
- **账号隔离**：下单时自动关联当前选中的账号，数据隔离
- **便捷切换**：用户可在 Header 快速切换账号，无需进入专门页面

### 3.2 账号类型

| 类型 | 说明 | 优先级 |
|------|------|--------|
| 实盘账号 | 真实券商账户，连接 QMT | 排第一 |
| 仿真账号 | 模拟交易账户 | 排第二 |

### 3.3 交互设计

#### 3.3.1 Header 用户下拉菜单

```

│ 已选择的账号:                   │

│   仿真账号A(仿真)               │
│   仿真账号B(仿真)               │

│ [账号管理]                      │

```

**设计要点**：
- 实盘账号前有 ★ 标记
- 账号后显示类别（实盘/仿真）
- 切换账号后，整个交易模块数据自动刷新

#### 3.3.2 Sidebar 菜单调整

**home.html（首页）**：
- 概览相关功能：概览、账号管理、交易记录、系统设置

### 3.4 数据关联

当用户选择某个账号后，整个交易模块的数据将与该账号关联：

| 数据项 | 关联内容 |
|--------|----------|
| 资产信息 | 该账号的总资产、现金、市值 |
| 持仓 | 该账号的持仓明细 |
| 委托 | 该账号的当日委托 |
| 下单 | 委托发送到该账号 |

## 四、关键设计决策

### 4.1 布局决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 最大宽度 | 1400px | 兼顾大屏显示和内容聚焦 |
| 资产展示 | 水平信息条 | 节省垂直空间，信息密度高 |
| 持仓表 | 表格形式 | 专业交易终端的通用做法 |

### 4.2 交互决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 股票搜索 | 代码/名称/拼音 | 符合用户习惯 |
| 价格输入 | 手动 + 快捷按钮 | 兼顾精确和效率 |
| 仓位选择 | 全仓/1/2/1/3/1/4 | 简化计算，一键满仓 |

### 4.3 视觉决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 买入按钮 | 红色 | 传统金融配色 |
| 卖出按钮 | 绿色 | 传统金融配色 |
| 涨停按钮 | 红色背景 | 强调风险 |
| 跌停按钮 | 绿色背景 | 强调风险 |
