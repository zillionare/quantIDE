## 组件详细设计

### 1. Header 区域
- 保持现有设计不变
- Logo + 主导航 + 用户信息

### 2. Sidebar 左列（三行结构）

#### 2.1 第一行：板块选择 Select
```
┌────────────────────┐
│ [板块选择 ▼]       │  ← Select 下拉框
│ test -- 自定义     │
└────────────────────┘
```

**Select 选项内容：**
```
┌────────────────────────────┐
│ test -- 自定义             │  ← 当前选中
├────────────────────────────┤
│ 钢铁行业 -- 行业           │
│ 新能源车 -- 概念           │
│ 深圳 -- 地区               │
│ 沪深300 -- 指数            │
│ ...                        │
├────────────────────────────┤
│ + 新建板块                 │  ← 最后一项，特殊样式
└────────────────────────────┘
```

**数据说明：**
- 板块名称：从数据库读取的板块名称
- 板块类型：自定义、行业、概念、地区、指数等
- 来源：包含用户自定义板块和 tushare 等外部数据源已同步的板块

**交互逻辑：**
- 点击 Select 展开下拉列表
- 选择板块后，第二行显示该板块的所有个股
- 选择"+ 新建板块"，弹出创建板块模态框
- 默认选中第一个板块

#### 2.2 第二行：板块个股列表
```
┌────────────────────┐
│ ▶ 平安银行         │  ← 选中状态（高亮）
│   000001.SZ        │
├────────────────────┤
│   万科A            │
│   000002.SZ        │
├────────────────────┤
│   国农科技         │
│   000004.SZ        │
├────────────────────┤
│   世纪星源         │
│   000005.SZ        │
├────────────────────┤
│   ...              │
└────────────────────┘
```

**交互逻辑：**
- 显示当前选中板块的所有个股
- 点击个股：在右侧K线列表中高亮显示该个股（滚动到可视区域）
- 默认选中第一个个股（仅高亮，不触发其他动作）
- 列表支持滚动（如果个股数量多）

#### 2.3 第三行：所属板块（选中个股后显示）
```
┌────────────────────┐
│ 所属板块:          │
├────────────────────┤
│ • 沪深300          │  ← 可点击
│ • 银行板块         │  ← 可点击
│ • 深证50           │  ← 可点击
└────────────────────┘
```

**显示时机：**
- 初始状态：隐藏或显示提示"选择个股查看所属板块"
- 点击第二行的某个个股后，显示该个股所属的所有板块

**交互逻辑：**
- 点击所属板块中的某个板块：
  - 第一行的 Select 切换到该板块
  - 第二行更新为该板块的个股列表
  - 被点击来源的个股在第二行中高亮显示
  - 右侧K线列表更新为该板块的所有个股

### 3. Main Content 右列

#### 3.1 工具条
```
┌─────────────────────────────────────────────────────────────────┐
│ [个股/指数 ▼] [搜索...] [日线][周线][月线][多周期 ▼]          │
└─────────────────────────────────────────────────────────────────┘
```

**组件说明：**
- **类型切换**（左侧）：`个股/指数 ▼` 下拉框
  - 选项：个股、指数
  - 影响搜索范围

- **搜索输入框**（中间）：
  - 宽度：约8个字符（w-32）
  - 支持模糊匹配
  - 输入时显示下拉建议列表

- **周期按钮**（右侧）：
  - `日线`、`周线`、`月线`：切换K线周期
  - `多周期 ▼`：下拉选择更多周期选项
  - 对所有显示的K线图同时生效

**搜索交互逻辑：**
1. 用户在搜索框输入时实时显示匹配建议
2. 选择个股后：
   - 左侧第三行"所属板块"区域显示该个股所属的所有板块
   - 右侧K线列表中，如果被搜索的个股在当前板块中，则高亮显示
   - 如果被搜索的个股不在当前板块中，提示用户"该个股不在当前板块，点击所属板块查看"

#### 3.2 单股票K线图列表
右侧主体区域以垂直列表形式展示当前板块的所有个股K线图，**每行只显示一支个股**：

```
┌──────────────────────────────────────────────────────────────────┐
│  平安银行 (000001.SZ)                               日线 周线 月线│
│  9.21  +0.05 (+0.54%)                                           │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │                                                          │   │
│  │                      [K线图]                             │   │
│  │                                                          │   │
│  │                      [成交量]                            │   │
│  └──────────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────────┘
```

**布局规格：**
- 垂直列表布局：每行一支个股
- 每个K线卡片宽度：100%（占满右列宽度）
- 每个K线卡片高度：约 400px
- 卡片间距：16px

**加载策略：**
- 初始加载：前10只股票的K线图
- 滚动加载：用户滚动到底部时，再加载接下来的10只
- 加载指示器：底部显示"加载中..."或"没有更多了"

**K线卡片内容：**
1. **标题栏**：股票名称 + 代码 + 当前价格 + 涨跌幅 + 周期切换按钮
2. **K线图**：蜡烛图 + MA均线（MA5/MA10/MA20/MA60）
3. **成交量图**：底部小图

**交互逻辑：**
- 点击卡片：可以放大查看或进入详情页（可选）
- 周期切换：可以设置为仅影响当前卡片，或全局同步所有卡片
- 鼠标悬停：显示十字光标和数据提示
- 高亮显示：当左侧选中个股时，对应的K线卡片边框高亮

## 状态流转

### 初始状态
1. 页面加载时：
   - 从数据库加载所有板块列表
   - 第一行 Select 默认选中第一个板块
   - 第二行显示该板块的所有个股，默认选中第一个
   - 第三行隐藏或显示提示
   - 右侧加载该板块前10只股票的K线图（每行一支）

### 切换板块（通过 Select）
1. 用户点击第一行 Select，选择某个板块
2. 第二行更新为新板块的个股列表，默认选中第一个
3. 第三行清空或显示提示
4. 右侧清空当前K线列表，加载新板块的前10只股票K线图

### 点击板块个股
1. 用户点击第二行的某个个股
2. 第三行显示该个股所属的所有板块
3. 右侧K线列表中，该个股对应的卡片高亮显示（滚动到可视区域）

### 点击所属板块
1. 用户点击第三行的某个所属板块
2. 第一行 Select 切换到该板块
3. 第二行更新为该板块的个股列表，被点击来源的个股高亮显示
4. 第三行更新为该个股在新板块中的所属板块（可能相同或不同）
5. 右侧K线列表更新为该板块的所有个股，被点击来源的个股高亮显示

### 搜索个股
1. 用户在工具条搜索框输入，选择个股
2. 如果该个股在当前板块中：
   - 第二行中该个股高亮显示
   - 第三行显示该个股所属板块
   - 右侧K线列表中该个股卡片高亮
3. 如果该个股不在当前板块中：
   - 显示提示"该个股不在当前板块"
   - 第三行显示该个股所属板块
   - 用户可以点击第三行的所属板块切换

### 切换周期
1. 用户点击工具条的周期按钮
2. 所有K线卡片同步切换到新周期
3. 重新加载对应周期的K线数据

## 技术实现要点

### API 需求
1. `GET /api/v1/sectors` - 获取所有板块列表（包含名称和类型）
2. `GET /api/v1/sectors/{id}/stocks` - 获取板块成分股（支持分页）
3. `POST /api/v1/sectors` - 创建新板块
4. `GET /api/v1/stocks/search?q={query}&type={type}` - 搜索个股/指数
5. `GET /api/v1/stocks/{symbol}/sectors` - 获取个股所属板块
6. `GET /api/v1/kline/{symbol}?start={start}&end={end}&freq={freq}` - 获取K线数据

### 前端状态
```typescript
interface AnalysisState {
  // 板块列表（第一行 Select 用）
  sectors: Sector[];

  // 当前选中的板块
  selectedSector: Sector | null;

  // 板块个股列表（第二行，分页加载）
  sectorStocks: Stock[];
  currentPage: number;
  hasMore: boolean;

  // 当前选中的个股（第二行点击）
  selectedStock: Stock | null;

  // 选中个股的所属板块（第三行）
  stockSectors: Sector[];

  // 搜索相关
  searchQuery: string;
  searchType: 'stock' | 'index';
  searchResults: Stock[];

  // K线图设置（全局）
  klineFreq: 'day' | 'week' | 'month';

  // K线数据（分页加载）
  klineStocks: Stock[];
  klinePage: number;
  klineHasMore: boolean;
}

interface Sector {
  id: string;
  name: string;
  type: 'custom' | 'industry' | 'concept' | 'region' | 'index';
}

interface Stock {
  symbol: string;
  name: string;
  price?: number;
  change?: number;
  changePercent?: number;
}
```

### 性能优化
1. **虚拟滚动**：如果板块个股数量很多，使用虚拟滚动优化
2. **K线图懒加载**：只加载可视区域内的K线图
3. **数据缓存**：缓存已加载的K线数据，切换周期时优先使用缓存
4. **防抖加载**：滚动加载时防抖处理，避免频繁请求
