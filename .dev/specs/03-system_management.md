# 系统维护

这个模块提供系统设置、数据维护等功能。UI 对应的路由是 `/system`。

## 导航

当用户在 header 中点击`系统维护`菜单时，sidebar 将加载以下菜单项：

```md
- 数据维护:
    - 交易日历
    - 股票列表
    - 行情数据
- 系统设置:
    - 定时任务
    - 交易网关
    - 数据源
```


## 交易日历

交易日历模块提供以下功能：

## 日历列表

显示交易日历并提供查找功能，用以帮助量化研究者判断某天是否为交易日。

界面如下图所示（草图）：

![](https://cdn.jsdelivr.net/gh/zillionare/imgbed2@main/images/2026/03/20260331184854.png)

在界面元素中，注意到以下各项：

1. 第一行右侧有一个立即更新按钮。点击后会调用数据 API 以更新日历。
2. 第二行是日历工具条。由左边的箭头组< > << >>及右对齐的日期选择器组成。当点击 < >按钮时，分别加载前后一个月的日历；当点击 << 或者 >> 时，分别加载前后一年的日历。当点击日期选择器时，以下拉框方式显示如2026/12这样的文本项。当用户选择后，下方的表格就显示该月的日历。
3. 第三行是日历表格，显示当前选择的月份的交易日历。如果某一天是休市，则以阴影背景、红色字体显示休市。否则显示日期。

在初始化时，显示当前月份（含日期选择器初始化）。

## 股票列表

量化研究员用来查询股票的基本信息。支持按代码、名称、拼音进行模糊查询。

界面示意如下：

![](https://cdn.jsdelivr.net/gh/zillionare/imgbed2@main/images/2026/03/20260331190847.png)

在界面元素中，注意到以下各项：

1. 第一行右侧有一个『立即更新』按钮。点击后，会调用数据接口 API 以更新股票列表。
2. 第2行是模糊查询输入框。当用户输入停顿250ms 后，就会触发后台查询，并返回匹配项并更新下面的列表
3. 第3行是当前匹配到的股票列表项。
4. 第4行（图中未显示）是分页。列出最多7个页码，一个直接输入并跳转的页码框，以及每页记录数设置（允许20， 50， 100）

## 行情数据

以表格的形式，显示原始的股票行情数据，供研究员查询和比对 。

![](https://cdn.jsdelivr.net/gh/zillionare/imgbed2@main/images/2026/03/20260331192123.png)

1. 支持按日期（起始日期-结束日期）、证券代码进行过滤。表格下方增加分页指标。

2. 默认以不复权方式显示。增加前复权按钮。当点击时，以结束日期为前复权起点，数据向前复权后显示。

3. 在没有任何过滤条件时，默认只返回首页数据。
