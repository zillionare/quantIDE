# 布局和导航

本文档规定了页面布局、导航及相关菜单项，以及它们之间的交互机制。

## 总体布局

UI 采用现代化的单页应用（SPA）架构，提供交易、策略管理、数据维护等核心功能。

采用经典的 Header + Sidebar + Main 三栏布局：

![](https://cdn.jsdelivr.net/gh/zillionare/imgbed2@main/images/2026/03/20260331162723.png)

header是页面顶端的一行。包含 (logo, brand), (一级导航)和(用户头像)三个区域。

sidebar 上显示二级菜单。它与一级导航联动。当一级导航条目变更时，二级菜单也跟着改变。

当二级菜单变化时，main content 区的内容一般也要跟着改变。

## 样式与风格

应用的主题色是 #e41815 。当使用该颜色作为背景时，主要字体将使用白色。该颜色只作为部分强调型按键的背景色或激活提示色，不宜作为 Header 等大面积区域的背景使用（Header 等可使用白色作为背景色）。

一般按钮使用 #fcfcfc 作为背景， #e41815 作为前景字体色，也可以使用 #2c3030 作为字体色。

当需要使用图标时，不使用 emoji 字符，也不使用 svg，而是使用全应用惟一的 icon 集。

## Header 设计

Header 从左到右依次为：

1. **Brand**: 系统名称"匡醍量化"
2. **空白区域**: 用于视觉平衡
3. **一级导航**: 水平排列的导航菜单
4. **alarm**: 显示为 alarm (bell) 图标，用以显示未读取的消息。当存在未读消息时，通过 overlay 来提示未读消息数。
5. **用户头像**: 仅显示头像本身
6. **下拉菜单指标**： 显示一个下拉箭头，当点击时，展开下拉菜单包含重设密码、退出登录等

## 一级导航菜单项

| 导航项   | 路由          | 说明               |
| -------- | ------------- | ------------------ |
| 策略     | `/strategy`   | 策略管理           |
| 系统维护 | `/system`     | 系统配置、数据维护 |
| 实盘     | `/live`       | 实盘交易界面       |
| 仿真     | `/papertrade` | 仿真交易界面       |

### 无 gateway 时的导航退化

当系统已经完成初始化，但 `gateway` 未启用时，一级导航需要进入降级模式：

1. “策略”和“系统维护”保持正常可访问。
2. “实盘”和“仿真”仍然保留在一级导航中，但点击时不直接进入交易页面，而是弹出提示对话框。
3. 提示文案应明确说明：当前尚未配置 gateway，因此无法使用实盘和仿真交易。
4. 对话框需要提供至少一个明确操作入口，用于前往交易网关配置页面。
5. 首页与其它非交易页面都不应再因为“没有交易账号”而强制弹出创建账号对话框；交易账号创建应只在用户主动进入交易入口后触发。
6. 当 `gateway` 不可用时，访问首页默认高亮“策略”一级导航，而不是“实盘”。

### 有 gateway 时的默认激活项

当用户已经完成 gateway 配置并成功登录后，默认激活的一级导航应为“实盘”：

1. 如果当前页面没有明确归属到其它一级导航，则顶部一级导航默认高亮“实盘”。
2. 在该默认状态下，侧边栏应联动展示“实盘”对应的二级菜单。
3. 该默认激活仅适用于 gateway 已可用的场景；如果 gateway 未启用，则仍保持此前的降级逻辑。


## 头像菜单

点击头像区域的下拉菜单图标时应当居中对齐展开。

菜单项包括：

1. (icon) 重设密码
2. (icon) 退出登录

当用户点击退出登录时，应该清空会话信息，跳转至登录界面。

## sidebar

sidebar 用来加载一级导航对应的二级菜单。二级菜单在各个模块的功能文档中具体定义，一般是 yaml 语法块，比如：

```yaml
- 数据维护:
    - 交易日历
    - 股票列表
    - 行情数据
- 系统设置:
    - 定时任务
    - 交易网关
    - 数据源
```

系统将通过 accordin 风格的组件来支持以上这种多级菜单。一般来说，多级菜单的第一级不直接关联路由。当用户点击它们时，需要展开、隐藏子菜单。

当 sidebar 切换后，如果没有明确说明或者重定向指示，内容区默认会显示每一个菜单项的内容。如果第一个菜单项是多级菜单，则展开该项并显示第一项。

## main area

main area 显示界面由各模块功能说明决定。但如非特别说明，都会在顶部显示一个 breadcrum (透明度为0.8)。
