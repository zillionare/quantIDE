# 布局和导航

本文档规定了页面布局、导航及相关菜单项，以及它们之间的交互机制。

## 总体布局

UI 采用现代化的单页应用（SPA）架构，提供交易、策略管理、数据维护等核心功能。

采用经典的 Header + Sidebar + Main 三栏布局：

![](https://cdn.jsdelivr.net/gh/zillionare/imgbed2@main/images/2026/03/20260331162723.png)

header是页面顶端的一行。包含 (logo, brand), (一级导航)和(用户头像)三个区域。

sidebar 上显示二级菜单。它与一级导航联动。当一级导航条目变更时，二级菜单也跟着改变。

当二级菜单变化时，main content 区的内容一般也要跟着改变。

## 样式与风格

应用的主题色是 #e41815 。当使用该颜色作为背景时，主要字体将使用白色。该颜色只作为 header, 部分强调型按钮（需要特别指出）的背景色，不宜大量使用。

一般按钮使用 #fcfcfc 作为背景， #e41815 作为前景字体色，也可以使用 #2c3030 作为字体色。

当需要使用图标时，不使用 emoji 字符，也不使用 svg，而是使用全应用惟一的 icon 集。

## Header 设计

Header 从左到右依次为：

1. **Brand**: 系统名称"匡醍量化"
2. **空白区域**: 用于视觉平衡
3. **一级导航**: 水平排列的导航菜单
4. **alarm**: 用以显示未读取的消息。当存在未读消息时，通过 overlay 来提示未读消息数。
5. **用户头像**: 显示当前登录用户，下拉菜单包含个人设置、退出登录等

## 头像菜单

在头像下通过 overlay 叠加一个小箭头，以提示此处有菜单项。

菜单项包括：

1. (icon) 重设密码
2. (icon) 退出登录

当用户点击退出登录时，应该清空会话信息，跳转至登录界面。

## sidebar

sidebar 用来加载一级导航对应的二级菜单。二级菜单在各个模块的功能文档中具体定义，一般是 yaml 语法块，比如：

```yaml
- 数据维护:
    - 交易日历
    - 股票列表
    - 行情数据
- 系统设置:
    - 定时任务
    - 交易网关
    - 数据源
```

系统将通过 accordin 风格的组件来支持以上这种多级菜单。一般来说，多级菜单的第一级不直接关联路由。当用户点击它们时，需要展开、隐藏子菜单。

当 sidebar 切换后，如果没有明确说明或者重定向指示，内容区默认会显示每一个菜单项的内容。如果第一个菜单项是多级菜单，则展开该项并显示第一项。

## main area

main area 显示界面由各模块功能说明决定。但如非特别说明，都会在顶部显示一个 breadcrum (透明度为0.8)。

## 4. 功能模块详细设计

### 4.1 登录与退出

#### 4.1.1 登录页面

暂不实现。

**路由**: `/login`

**功能**:
- 用户名/密码登录
- 记住登录状态（可选）
- 登录失败提示

**UI 元素**:
- Logo 和系统名称
- 用户名输入框
- 密码输入框
- 登录按钮
- 错误提示区域

**流程**:
1. 用户输入凭证
2. 系统验证用户身份
3. 成功后跳转至首页（或之前访问的页面）
4. 失败显示错误信息

#### 4.1.2 退出登录

**触发方式**: Header 用户菜单中的"退出"按钮

**流程**:
1. 清除会话信息
2. 跳转至登录页面

### 4.2 交易模块

**一级路由**: `/trade`

#### 4.2.1 二级菜单

| 菜单项 | 路由                | 说明             |
| ------ | ------------------- | ---------------- |
| 仿真   | `/trade/simulation` | 仿真交易账户管理 |
| 实盘   | `/trade/live`       | 实盘交易账户管理 |

#### 4.2.2 仿真交易页面

**路由**: `/trade/simulation`

**功能列表**:

1. **账户列表展示**
   - 显示所有仿真账户（Portfolio）
   - 字段：账户ID、名称、创建时间、状态、总资产、收益率
   - 支持搜索、筛选、排序

2. **创建仿真账户**
   - 弹出对话框
   - 输入项：
     - 账户名称（必填）
     - 初始资金（默认 1,000,000）
     - 佣金费率（默认 0.0001）
     - 描述信息（可选）
     - 持仓市值更新间隔（默认 10 秒，可选 3/5/10 秒）

3. **账户详情**
   - 点击账户进入详情页
   - 显示：
     - 账户基本信息
     - 当前持仓列表（资产代码、数量、成本、市值、盈亏）
     - 可用资金
     - 当日订单列表
     - 成交记录

4. **交易操作**
   - 买入：输入资产代码、数量/金额、价格（0 为市价）
   - 卖出：输入资产代码、数量/比例、价格
   - 撤单：取消未成交订单

5. **账户控制**
   - 启动/停止账户
   - 删除账户（需确认）

### 4.2.3 实盘交易页面

**路由**: `/trade/live`

**功能**:
- 与仿真交易类似，但连接真实交易柜台（QMT）
- 增加风险提示
- 显示柜台连接状态

**额外功能**:
- 柜台配置：设置 QMT 连接参数
- 同步状态：从柜台同步持仓、资金、订单

### 4.3 我的策略模块

**一级路由**: `/strategy`

#### 4.3.1 二级菜单

| 菜单项   | 路由             | 说明               |
| -------- | ---------------- | ------------------ |
| 策略列表 | `/strategy/list` | 所有策略的管理入口 |

#### 4.3.2 策略列表页面

**路由**: `/strategy/list`

**功能列表**:

1. **策略列表展示**
   - 显示所有策略文件/目录
   - 字段：策略名称、类型、创建时间、最后修改时间、回测次数
   - 支持搜索、筛选

2. **策略操作**
   - **查看**: 在浏览器中查看策略文件内容
   - **编辑**: 打开 VSCode 编辑策略（需要配置 VSCode 路径）
   - **回测**: 弹出回测配置对话框
   - **删除**: 删除策略文件（需确认）

3. **创建新策略**
   - 基于模板创建
   - 或上传策略文件
